Question 1 (10 marks)

In the q1 directory, you will find the following files/directories:

  • countEven.c: the only file you need to modify and submit. You will also find the supporing header file countEven.h
  • autotest: simple auto testing script, important:
    • This script only tests simple cases, you need to extensively test your program for a wide range of test cases.
    • It is possible that this script may not be able to properly handle your program. You need to manually test your program (if required, without autotest script) in order to make sure that it satisfies the requirements.
  • BST.h and BST.o: an implementation of a simple binary search tree ADT, similar to (but not the same as) the one you used in your lab exercises. Please open and read BST.h for the available functions for this question. Please note that BST.c is NOT available for this exam.
  • testCountEven.c: the program to test your function. The program reads tree data (and other inpt data) from stdin, calls the function you implement countEven, and outputs the answer to stdout. You must carefully study how your function is tested by testCountEven.c. You can find out more about the behaviour of the testCountEven program by looking at the files testCountEven.c, autotest and the files in the tests directory. The files named tX.in are input files for the test program testHasPath and the corresponding expected output are in tX.exp.
  • Makefile: to build the required files.
  • tests: the directory containing input (*.in) and expected output (*.exp) files for few simple test cases.

Your task for this question is to implement the countEven(BST t) function in the file countEven.c.

  • The countEven(BST t) function takes one argument: a binary search tree t, and counts the number of keys that are even (nodes with key values that are even) in a given tree, and returns the count. 
int countEven(BST t);

You can compile and test your function using the following commands: 

$ make    # builds the required program testCountEven
$ ./autotest  # apply all the tests in the tests directory to the program testCountEven
$ ./autotest <test-number>   # apply only one given test case from the tests directory to the program testCountEven

You can find out more about the behaviour of the testCountEven program by looking at the files testCountEven.c, autotest and the files in the tests directory. The files named tX.in are input files for the test program testCountEven and the corresponding expected output are in tX.exp.

If you want to run say test case 2, use the following command: 

$ ./autotest 2

If you want to run all the sample test cases provided, use the following command: 

$ ./autotest

After you run the tests, additional files will appear in the tests directory: tX.out contains the output (from your function) of running test tX.

IMPORTANT: You can add debugging code to countEven.c or testCountEven.c if you want, however, make sure that you remove it before testing and submitting, otherwise your output won't match the expected output and you'll fail all the tests. You can also add any auxiliary functions to countEven.c that you think are necessary.

Once you are satisfied with your program, submit it using the command: 

$ submit q1

This will make a copy of the countEven.c file from the q1 directory as your answer for this question. You can run the submit command as many times as you like, but make sure that your final submission compiles without any errors or warnings.

Testing:

Test your program thoroughly, possibly using test cases additional to those supplied. Your program will be tested using test cases different to the example test cases provided in the tests directory.

If you guess a possible answer/output and pass a test, you will not be awarded marks. Your program must properly solve the task discussed above.
